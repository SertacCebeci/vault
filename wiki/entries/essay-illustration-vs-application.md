# Essay — Illustration vs. Application

## Lead

The decision-tree boundary between [[lens-illustration]] (priority 20) and [[lens-application]] (priority 30) is fuzzier than the criteria suggest. An "application" with a single concrete subject — one named teacher's morning routine, one specific study program — often reads as an illustration that happens to include actionable steps. This essay proposes a rule of thumb to keep the categories cleanly separated.

## The fuzziness

[[lens-illustration]] criteria: a story with protagonist, setting, outcome.

[[lens-application]] criteria: a practitioner could follow this as instructions.

In practice, many entries satisfy both: a teacher (protagonist) running a particular routine (setting) that produced a measurable outcome (outcome) — and the routine is followable. Which lens fires first wins, but the criteria above do not produce a stable answer because both clearly match.

## Proposed rule of thumb

If the entry's primary work is **describing a singular instance** to anchor a more general principle, it is an `illustration` — even if a reader could imitate the instance.

If the entry's primary work is **specifying a procedure** that could be applied across instances (with named parameters, prerequisites, success criteria), it is an `application` — even if it includes one named example as motivation.

Test: ask "if I changed the protagonist and setting, would this entry still serve its purpose?"

- If yes (the entry generalizes), it is an `application` — illustrations *would not* survive that change.
- If no (the entry is *about* this specific case), it is an `illustration`.

Where the boundary is genuinely 50/50, prefer `illustration` because it is at the earlier priority — and `applies:` (in an application's frontmatter) tends to be a more demanding field to complete than `illustrates:` (in an illustration's frontmatter).

## Counterargument

The decision tree is supposed to be deterministic — adding a "rule of thumb" essay subverts the priority order. If the lens criteria are genuinely fuzzy, the right move is to amend the lens criteria themselves rather than route around them with editor judgement.

The response: the essay is a description of how editors are already deciding, not a proposed amendment. Once the pattern is well-attested, the lens criteria can be amended directly via [[lens-application]] and [[lens-illustration]] edits.

## Status

Essay. Binds nothing. Watch for ingestion runs that produce illustrations-or-applications and see whether the rule-of-thumb's classifications track later editorial agreement.
