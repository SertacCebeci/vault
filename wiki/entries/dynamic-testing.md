# Dynamic Testing

Dynamic testing is an assessment approach that treats performance as a moving point on a developmental path rather than as a fixed verdict about ability. Instead of using a test only to rank learners, it uses a repeating cycle of test, targeted improvement, and retest to identify weaknesses, guide practice, and measure growth in expertise over time.

## Elaboration

Dynamic testing offers an alternative to static aptitude measures and to learning-styles labeling. A low score is interpreted not as "this is what you are" but as "this is where you need work." That shift matters because it directs attention toward the next learning move rather than toward an identity claim.

The model has three steps. First, some form of test or demanding experience shows where performance breaks down. Second, the learner focuses effort on those weak spots using techniques that actually build competence, such as retrieval practice, reflection, spacing, and feedback. Third, the learner tests again to see what improved and what still needs work. This makes assessment part of learning rather than a ritual that happens after learning is supposedly finished.

Dynamic testing also bridges two major threads already in the wiki. It extends [[calibration]] by turning feedback into an ongoing loop instead of a one-time correction, and it fits the logic of [[desirable-difficulty]] because setbacks become informative only when they trigger productive effort. In that sense, Bruce Hendry's career in the chapter is a life-sized example: each failed investment teaches him what to ask next and sharpens the model he brings to the next opportunity.
